X3 可利用BUG合集

2018-03-22

Original post on deeptimes. 

以下内容均为本人凭记忆手打，如有出错遗漏欢迎各位大佬指出。
所有BUG基于原版(Vanilla)，因为修改版（Modified）作弊可以直接用脚本编辑器调出，不需要利用卡BUG这种低阶方式。所有列出的BUG不会导致Modified Tag。
鉴于LZ本人只玩过TC2.5, AP3.3, 所以下文的适用版本号都是凭印象给的，基本准(cuo)确(wu)。当然如果你看到了Lv.6-7, 你会发现神马版本号都是浮云。

正式开始前请确保您已年满18……请确保观看本合集时有家长陪同（误）

警告：卡BUG属于作弊，小作怡情，大作伤身，强作灰飞烟灭
按Bug强度从低到高排列，如有不适及时退场

======IMBA分割线 Lv.0 SL大法======

SL大法。AutoHotKey。
本人刷了近9000个存档获得了250.3m/s的Hyperion Vanguard，配合散落宇宙的Crate加速到了372m/s，超过了Spring Blossom。
TC中可用SL保护登陆士兵。

======IMBA分割线 Lv.1 无敌舰船======

获取无敌舰船
限制：X3TC，2.5及以下。2.6后获取舰船自动取消无敌，但之前版本获得的舰船无敌Tag不会消失。3.0后之前版本获得的无敌舰船自动取消无敌Tag.
#deca cpu ship
限X3TC Terren Plot
传说中的大筒子,获取方法参见置顶帖
可装Gamma Kyon Emitter 的无敌M1
Pirate Truelight Seeker
限X3TC Goner Plot
最终完成任务前强行登陆
可装Gauss Canon的无敌M6
Pirate Blastclaw Prototype
限X3TC Goner Plot
在空间站爆炸前完成任务，然后Claim
巨大的650货舱的无敌M3

======IMBA分割线 Lv.2 无限购买======

船厂BUG
X3TC：每次使用必须购买>2条船
X3AP：只需将完全相同的>2条船停靠在船厂即可
只要该星区装备坞本身存在的武器，护盾，导弹即可复制。
具体操作：在多条船（x条）同时装备界面，同时购入，全部卖出，你会发现即使原本只有一个你也可以买入，卖出后总共获得x个
因此，必须装备坞天然存在该武器并且储量>0时方可使用本Bug。对于FAA，IBL等不能用此方法获得，因为并没有装备坞存储他们。
购买会扣钱。

======IMBA分割线 Lv.3 刷钱======

刷股票BUG
这本身来说不是一个BUG，但是一个BUG一样的机制。
以Energy Bolt Chain Gun为例，单体价格15w，体积1.
Spring Blossom或Hyperion Vanguard轻松携带1000个，购买2000股，一次操纵可盈利7000w
若使用Aran的120000货舱，单次操纵可获得约80亿（本金约300亿）
Aran可以套Aran，或者使用多条船，因此该BUG实际上具有在内存限制以内刷钱的无限潜力。
EBCG可以配合船厂Bug等手段很方便的获取。

======IMBA分割线 Lv.4 免费复制======

Freight Drone Bug，货运无人机Bug
限制：TC3.1以下，AP2.5以下（不包括）
包含至少两个BUG

BUG1免费复制任意数量任意物品
对货运无人机使用Transfer命令，将A舰货舱已有的某商品WareA传输至B舰，数量-x。
B舰什么都不会发生，A舰货舱瞬间+x件WareA。
玩家钱不会扣。
可用来获得几乎所有已经有但数量不足的商品，而且完全免费。潜力无限。甚至
包括EBCG。
包括FAA，IBL这种无法利用船厂BUG购买的产品。
包括Kyon Emitter。
包括Repair Laser。前提是你已经有了一个。
包括PHQ。
…………
不能用此方法获得货运无人机。

BUG2获取LaserTower内置武器
命令Freight Drone从Laser Tower获取TBC即可。
获取了还能捡回Laser Tower重新部署继续刷。

======IMBA分割线 Lv.5 创造======

Menu Shift BUG，菜单转移Bug
限制：TC3.2以下，AP3.3以下。
可创造能出现在Equipment Dock但数量为0的商品，例如Repair Laser。
可复制任意商品。（我没测试）
甚至可复制Engine Tunning！（不好意思我也没测试）
例：
以创造Repair Laser为例
利用装有CLS的船在玩家装备坞、Xenon HUB、PHQ等处。
具体操作：
宇航员停靠在Dock，利用Dock的命令Ware Manager即可添加Repair Laser至列表。
A船软件齐全。包括CLS，Trade Command Software MK I 等等。
A船装WareP。例如大容量护盾，1个即可。必须是A船本身不能安装的。必须是装备坞中本来没有的
A船装WareD。例如能量电池，数量任意。
你在C船。
对A船使用CLS命令，创建Waypoint如下
对B船Unload WareP
对B船Unload WareD
从B船Load WareD
对B船Unload WareD
从B船Load WareD
对B船Unload WareD
从B船Load WareD
对B船Unload WareD
从B船Load WareD
对B船Unload WareD
从B船Load WareD
……….
对B船Load WareP
中间循环次数取决于你的手速，手速不够多来几次，3-5次差不多了。
A船停靠在装备坞。
B船停靠在装备坞。
命令A船开始CLS。
从B船打开停靠的装备坞的管理界面，你会发现菜单行数周期性变化，而光标位置不变。
WareD应当处于价格列表第一个，
Repair Laser应当处于商品列表的最后一个。
光标移到WareD的价格调整，按Home
菜单下移后光标自动挪到Repair Laser的数量上，这时按Enter。
Repair Laser的数量会变为WareD的价格差。

======IMBA分割线 Lv.6 BUG恢复术======

X3利用脚本的签名认证来区别原生脚本和第三方脚本。
BUG被版本更新封堵了？不存在的。
使用游戏早期的带有BUG的脚本替换现有脚本，游戏依然认为签名有效。这样即可在后期版本中继续使用BUG。例如!ship.cmd.freightransfer.pre即为FreightTransfer Bug所依赖的脚本。
不能恢复Menu Shift BUG，因为BUG涉及窗口句柄而非脚本。

======IMBA分割线 Lv.7 创世======

X3利用脚本的签名认证来区别原生脚本和第三方脚本。
使用修改的advapi32.dll强行认证所有签名，可以在Vanilla下修改任意原生脚本，或运行任意第三方脚本。无限潜力。
不能运行需要进入Script Editor手动操作的脚本，因为进入Script Editor就会增加Modified Tag。但可以修改这些脚本的触发方式来避免。
工具及教程：

必须Win7。经检验，虚拟机可。
